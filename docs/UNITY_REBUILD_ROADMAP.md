# Shelter 2250 — Unity 재제작 로드맵(지휘용)

## A. 전환 전 “결정 5개”(오늘 30분)

1. Unity 버전: 최신 LTS
2. 플랫폼 1순위: PC/Mac(후에 WebGL)
3. 렌더/2D: `URP 2D` (라이트/연출 여지)
4. UI: `UGUI`(Canvas)로 빠르게 가고, 후에 필요하면 UI Toolkit 혼합
5. 목표: **Feature parity first** (재밌는 루프 확보 후 리팩토링)

---

## B. 2주 플랜(최소 리스크, 최대 전진)

### Week 1 — “Simulation Only” 완성

**목표**: 연출 없이도 ‘게임이 굴러감’을 증명.

- Day 1: Unity 프로젝트 생성/커밋 + 폴더 구조 고정
  - `Unity/` 디렉토리(또는 별도 repo) 결정
  - `.gitignore` Unity 항목 확인
- Day 2: `GameState` + `SaveService`(JSON) 구현
- Day 3: `GameTickService`(1초 틱) + `TimeScale(1/2/6)` + 게임 시간(분→시→일) 구현
- Day 4: `ResourceSystem` 구현(생산/소비/패널티) + 로그 HUD
- Day 5: `RoomTypeDefinition(SO)` + `ShelterGrid` + 배치 검증 룰(웹 스펙을 기준) 구현
- Day 6: `TraitDefinition(SO)` + `DwellerSystem`(상태/효율/틱) 구현
- Day 7: 패리티 체크(7일 시뮬) + 저장/불러오기 회귀 확인

**Week 1 DoD**
- 60초 실시간에 1/2/6시간 게임 시간이 정확히 진행
- “식수/식량이 끊기면 7일 이내 사망” 목표에 맞게 수치 조정 가능(데이터로)
- 저장 후 재실행 시 상태가 깨지지 않음

### Week 2 — “Minimal UI + Tutorial” 완성

**목표**: 신규 유저가 첫 성공(첫 3일 생존)을 경험.

- Day 8: HUD(자원/시간/배속/캡 수입) + 기본 패널(건설/거주민) UI 구축
- Day 9: Build mode(미리보기/배치 불가 이유 툴팁/비용 표시)
- Day 10: Room modal(업그레이드/배치/치료/연구 버튼의 훅만)
- Day 11: Dweller panel(배치/상태 바) + 간단 필터
- Day 12: 튜토리얼 v0
  - 시작 프롬프트: “튜토리얼 진행?”
  - 5단계: 발전실 → 정수실 → 식당 → 거주공간 → 창고/의료
  - 각 단계 완료 조건 + UI 하이라이트/안내 텍스트
- Day 13~14: 튜토리얼로 3일 생존 플레이테스트 + 수치 튜닝

**Week 2 DoD**
- 튜토리얼 ON: “첫 3일”을 안내 따라 안정적으로 생존
- 튜토리얼 OFF: 자유 플레이 가능(막히더라도 원인 UI로 노출)

## C. 백로그 우선순위(지휘 기준)

### P0 (없으면 게임이 아님)
- 시뮬레이션/저장/배속/배치/자원/거주민
- 튜토리얼 최소 버전

### P1 (재미의 최소 루프)
- 이벤트 5~10개(위기/기회)
- 탐사 1종(성공/실패/방사능/보상)
- 경제 지출처(부품/금속/의료/연구의 “써야 하는 이유”)

### P2 (연출/완성도)
- 룸 비주얼, 조명, 애니메이션
- 사운드/피드백
- 국가별 변주/장기 성장

## D. 작업 규율(재제작에서 제일 중요한 룰 6개)

1. 숫자/정의는 코드가 아니라 데이터(SO)로 이동
2. UI는 상태를 “그려주기만” 하고, 명령(Command)만 호출
3. 저장 데이터에는 참조가 아니라 ID만 저장
4. 틱 기반 시뮬레이션은 결정론(재현 가능) 유지
5. “새 기능”은 반드시 DoD/지표를 포함한 티켓으로만 진행
6. 연출은 Phase 3 이후(지금은 금지)
